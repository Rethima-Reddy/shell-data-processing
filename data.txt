<p><b>cURL</b> (pronounced 'curl'<sup id="cite_ref-FAQ,_haxx.se_4-0" class="reference"><a href="#cite_note-FAQ,_haxx.se-4">&#91;4&#93;</a></sup>) is a computer software project providing a <a href="/wiki/Library_(computing)" title="Library (computing)">library</a> (<b>libcurl</b>) and <a href="/wiki/Command-line" class="mw-redirect" title="Command-line">command-line</a> tool (<b>curl</b>) for transferring data using various network <a href="/wiki/Protocol_(computing)" class="mw-redirect" title="Protocol (computing)">protocols</a>.  The name stands for "Client <a href="/wiki/URL" title="URL">URL</a>",<sup id="cite_ref-Stenberg,_haxx.se,_2018_5-0" class="reference"><a href="#cite_note-Stenberg,_haxx.se,_2018-5">&#91;5&#93;</a></sup> which was first released in 1997.<sup id="cite_ref-RelFirst_6-0" class="reference"><a href="#cite_note-RelFirst-6">&#91;6&#93;</a></sup>
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CURL&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>cURL was first released in 1997.<sup id="cite_ref-RelFirst_6-1" class="reference"><a href="#cite_note-RelFirst-6">&#91;6&#93;</a></sup> It was originally named <i>httpget</i> and then became <i>urlget</i> before adopting the current name of cURL.<sup id="cite_ref-cURL_changelog,_2020_7-0" class="reference"><a href="#cite_note-cURL_changelog,_2020-7">&#91;7&#93;</a></sup><sup id="cite_ref-Stenberg,_Haxx_SE,_2020_8-0" class="reference"><a href="#cite_note-Stenberg,_Haxx_SE,_2020-8">&#91;8&#93;</a></sup> The original author and lead developer is the Swedish developer <a href="/wiki/Daniel_Stenberg" title="Daniel Stenberg">Daniel Stenberg</a>, who created cURL because he wanted to automate the fetching of currency exchange rates for <a href="/wiki/Internet_Relay_Chat" title="Internet Relay Chat">IRC</a> users.<sup id="cite_ref-17yearhistory_1-1" class="reference"><a href="#cite_note-17yearhistory-1">&#91;1&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="libcurl">libcurl</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CURL&amp;action=edit&amp;section=2" title="Edit section: libcurl">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>libcurl is a free client-side URL transfer <a href="/wiki/Library_(computing)" title="Library (computing)">library</a>,<sup id="cite_ref-BlueOne_9-0" class="reference"><a href="#cite_note-BlueOne-9">&#91;9&#93;</a></sup> supporting <a href="/wiki/Cookie_(web)" class="mw-redirect" title="Cookie (web)">cookies</a>, <a href="/wiki/DICT" title="DICT">DICT</a>, <a href="/wiki/FTP" class="mw-redirect" title="FTP">FTP</a>, <a href="/wiki/FTPS" title="FTPS">FTPS</a>, <a href="/wiki/Gopher_(protocol)" title="Gopher (protocol)">Gopher</a>, <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP/1</a><sup id="cite_ref-Stenberg,_GitHub,_2019_10-0" class="reference"><a href="#cite_note-Stenberg,_GitHub,_2019-10">&#91;10&#93;</a></sup> (with <a href="/wiki/HTTP/2" title="HTTP/2">HTTP/2</a> support), <a href="/wiki/HTTP_POST" class="mw-redirect" title="HTTP POST">HTTP POST</a>, <a href="/wiki/HTTP_PUT" class="mw-redirect" title="HTTP PUT">HTTP PUT</a>, <a href="/wiki/HTTP_proxy" class="mw-redirect" title="HTTP proxy">HTTP proxy</a> tunneling, <a href="/wiki/HTTPS" title="HTTPS">HTTPS</a>, <a href="/wiki/IMAP" class="mw-redirect" title="IMAP">IMAP</a>, <a href="/wiki/Kerberos_(protocol)" title="Kerberos (protocol)">Kerberos</a>, <a href="/wiki/LDAP" class="mw-redirect" title="LDAP">LDAP</a>, <a href="/wiki/POP3" class="mw-redirect" title="POP3">POP3</a>, <a href="/wiki/RTSP" class="mw-redirect" title="RTSP">RTSP</a>, <a href="/wiki/Secure_copy" title="Secure copy">SCP</a>, <a href="/wiki/SMTP" class="mw-redirect" title="SMTP">SMTP</a>, and <a href="/wiki/Server_Message_Block" title="Server Message Block">SMB</a>. The library supports the <a href="/wiki/File_URI_scheme" title="File URI scheme">file URI scheme</a>, <a href="/wiki/SSH_File_Transfer_Protocol" title="SSH File Transfer Protocol">SFTP</a>, <a href="/wiki/Telnet" title="Telnet">Telnet</a>, <a href="/wiki/Trivial_File_Transfer_Protocol" title="Trivial File Transfer Protocol">TFTP</a>, file transfer resume, FTP uploading, HTTP form-based upload, HTTPS certificates, LDAPS, proxies, and user-plus-password authentication.<sup id="cite_ref-curl1_11-0" class="reference"><a href="#cite_note-curl1-11">&#91;11&#93;</a></sup>
</p><p>The libcurl library is portable. It builds and works identically on many platforms, including <a href="/wiki/AIX" class="mw-redirect" title="AIX">AIX</a>, <a href="/wiki/AmigaOS" title="AmigaOS">AmigaOS</a>, <a href="#cite_note-Blackberry,_GitHub-12">&#91;12&#93;</a></sup> <a href="/wiki/OpenVMS" title="OpenVMS">OpenVMS</a>, <a href="/wiki/Darwin_(operating_system)" title="Darwin (operating system)">Darwin</a>, <a href="/wiki/DOS" title="DOS">DOS</a>, <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a>, <a href="/wiki/HP-UX" title="HP-UX">HP-UX</a>, <a href="/wiki/HURD" class="mw-redirect" title="HURD">HURD</a>, <a href="/wiki/IOS" title="IOS">iOS</a>, <a href="/wiki/IRIX" title="IRIX">IRIX</a>, <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/MacOS" title="MacOS">macOS</a>, <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a>, <a href="/wiki/NetWare" title="NetWare">NetWare</a>, <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a>, <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a>, <a href="/wiki/Symbian" title="Symbian">Symbian</a>, <a href="/wiki/Tru64" class="mw-redirect" title="Tru64">Tru64</a>, <a href="/wiki/Ultrix" title="Ultrix">Ultrix</a>, <a href="/wiki/UnixWare" title="UnixWare">UnixWare</a>, and <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>.<sup id="cite_ref-Microsoft_Techcommunity_13-0" class="reference"><a href="#cite_note-Microsoft_Techcommunity-13">&#91;13&#93;</a></sup>
</p><p>The libcurl library is free, <a href="/wiki/Thread-safe" class="mw-redirect" title="Thread-safe">thread-safe</a> and <a href="/wiki/IPv6" title="IPv6">IPv6</a> compatible. <a href="/wiki/Language_binding" title="Language binding">Bindings</a> are available for more than 40 <a href="/wiki/Computer_languages" class="mw-redirect" title="Computer languages">languages</a>, including <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>/<a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a href="/wiki/PHP" title="PHP">PHP</a> and <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>.
</p><p>The libcurl library can support axTLS,<sup id="cite_ref-axTLS,_SourceForge_14-0" class="reference"><a href="#cite_note-axTLS,_SourceForge-14">&#91;14&#93;</a></sup> <a href="/wiki/GnuTLS" title="GnuTLS">GnuTLS</a>, <a href="/wiki/Mbed_TLS" title="Mbed TLS">mbed TLS</a>, <a href="/wiki/Network_Security_Services" title="Network Security Services">NSS</a>, <a href="/w/index.php?title=QSOSSL&amp;action=edit&amp;redlink=1" class="new" title="QSOSSL (page does not exist)">QSOSSL</a> on <a href="/wiki/IBM_i" title="IBM i">IBM i</a>, <a href="/wiki/Security_Support_Provider_Interface" title="Security Support Provider Interface">SChannel</a> on Windows, <a href="/wiki/Secure_Transport" class="mw-redirect" title="Secure Transport">Secure Transport</a> on macOS and iOS, <a href="/wiki/Transport_Layer_Security" title="Transport Layer Security">SSL/TLS</a> through <a href="/wiki/OpenSSL" title="OpenSSL">OpenSSL</a>, and <a href="/wiki/WolfSSL" title="WolfSSL">wolfSSL</a>.
</p>
<h2><span class="mw-headline" id="cURL">cURL</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CURL&amp;action=edit&amp;section=3" title="Edit section: cURL">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>cURL is a command-line tool for getting or sending data including files using <a href="/wiki/URL" title="URL">URL</a> syntax. Since cURL uses libcurl, it supports every protocol libcurl supports.<sup id="cite_ref-curl1_11-1" class="reference"><a href="#cite_note-curl1-11">&#91;11&#93;</a></sup>
</p><p>cURL supports <a href="/wiki/HTTPS" title="HTTPS">HTTPS</a> and performs SSL certificate verification by default when a secure protocol is specified such as HTTPS.  When cURL connects to a remote server via HTTPS, it will obtain the remote server certificate, then check against its <a href="/wiki/CA_certificate" class="mw-redirect" title="CA certificate">CA certificate</a> store the validity of the remote server to ensure the remote server is the one it claims to be.  Some cURL packages are bundled with CA certificate store file.  There are several options to specify a CA certificate such as <tt>--cacert</tt> and <tt>--capath</tt>. The <tt>--cacert</tt> option can be used to specify the location of the CA certificate store file.  In the Windows platform, if a CA certificate file is not specified, cURL will look for a CA certificate file name “curl-ca-bundle.crt” in the following order:
</p>
<ol><li>Directory where the cURL program is located.</li>
<li>Current working directory.</li>
<li>Windows system directory.</li>
<li>Windows directory.</li>
<li>Directories specified in the %PATH% environment variables.<sup id="cite_ref-SSL,_haxx.se_15-0" class="reference"><a href="#cite_note-SSL,_haxx.se-15">&#91;15&#93;</a></sup></li></ol>
<p>cURL will return an error message if the remote server is using a self-signed certificate, or if the remote server certificate is not signed by a CA listed in the CA cert file. <tt>-k</tt> or <tt>--insecure</tt> option can be used to skip certificate verification.  Alternatively, if the remote server is trusted, the remote server CA certificate can be added to the CA certificate store file.
</p>
<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CURL&amp;action=edit&amp;section=4" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Basic use of cURL involves simply typing <tt>curl</tt> at the command line, followed by the URL of the output to retrieve:
</p>
<pre>   curl www.example.com
</pre>
<p>cURL defaults to displaying the output it retrieves to the standard output specified on the system (usually the terminal window).  So running the command above would, on most systems, display the www.example.com source-code in the terminal window. The -o flag can be used to store the output in a file instead:
</p>
<pre>   curl -o example.html www.example.com
</pre>
<p>More options that change the tool's behavior are available.
</p>
